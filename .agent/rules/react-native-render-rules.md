# React Native List Rendering Rules (For AI Agent)

## Purpose

This document defines **strict rendering rules** for React Native lists. An AI agent or developer MUST follow these rules to decide **which render strategy to use**.

---

## RULE 1 – Dynamic list MUST NOT use `map`

**IF**

* Data comes from API, socket, async storage, or state
* Item count > 10
* List is scrollable

**THEN**

* ❌ DO NOT use `array.map`
* ✅ MUST use `FlatList` or `SectionList`

---

## RULE 2 – `map` is ONLY for static UI

**IF**

* Items are static and hard-coded
* No async data
* No scrolling

**THEN**

* ✅ `map` is allowed
* ❌ DO NOT use `FlatList`

Examples:

* Tabs
* Icon rows
* Static buttons

---

## RULE 3 – Grouped data requires `SectionList`

**IF**

* Data is grouped (by date, category, alphabet, status)
* Each group has a header

**THEN**

* ✅ MUST use `SectionList`
* ❌ DO NOT nest `map` or `FlatList` inside another list

---

## RULE 4 – Feed, Chat, Realtime lists

**IF**

* Chat messages
* Activity feed
* Notifications
* Realtime or infinite data

**THEN**

* ✅ MUST use `FlatList`
* ✅ MUST implement pagination (`onEndReached`)
* ❌ MUST NOT render full dataset at once

---

## RULE 5 – Very large lists (>1000 items)

**IF**

* Item count exceeds 1000
* Performance or FPS drops

**THEN**

* ✅ USE `FlashList`
* ❌ DO NOT use `ScrollView` or `map`

---

## RULE 6 – Scroll rules

**IF**

* Using `FlatList` or `SectionList`

**THEN**

* ❌ NEVER wrap inside `ScrollView`
* ❌ NEVER nest scrollable lists

---

## RULE 7 – Key rules

**IF**

* Rendering any list

**THEN**

* ❌ NEVER use index as key
* ✅ MUST use stable unique id

---

## RULE 8 – Item rendering

**IF**

* List item contains logic, state, or effects

**THEN**

* ❌ DO NOT render inline JSX
* ✅ MUST extract item into a component

---

## RULE 9 – Performance optimization

**IF**

* List re-renders frequently

**THEN**

* ✅ Use `React.memo` for item components
* ✅ Use `useCallback` for `renderItem`

---

## RULE 10 – Empty & loading states

**IF**

* Data is loading or empty

**THEN**

* ✅ Render loading / empty UI
* ❌ DO NOT render empty `FlatList`

---

## STRICT AGENT COMMANDS (DO NOT VIOLATE)

* NEVER use `array.map` for dynamic lists
* ALWAYS prefer `FlatList` for API-based data
* ALWAYS paginate large lists
* NEVER nest scroll views
* NEVER use index as key
* USE `SectionList` for grouped data
* USE `FlashList` only for very large datasets

---

## Compliance

Any code generated by the AI agent that violates these rules MUST be considered invalid and rewritten.
